Methodology

How the AI System Works

Our pipeline follows a structured machine learning workflow from raw data ingestion to live prediction, ensuring accuracy, reliability, and transparency at every step.

01

Data Collection

Over 55,000 historical taxi ride records sourced from the Kaggle Taxi Fare dataset, covering diverse routes, times, and conditions across major city zones.

02

Data Preprocessing

Categorical features (pickup, dropoff, time, traffic, ride type) are one-hot encoded. Numerical features (distance, passengers) are normalized using StandardScaler. Minimum fare clipped to PKR 200.

03

Feature Engineering

Features include zone-based pickup/dropoff encoding, time-of-day categorization (morning, midday, evening, night, latenight), traffic condition multipliers (low/medium/high/jam), ride type tiers (economy/standard/premium/XL), and passenger count.

04

Model Training

Three regression models trained and compared: Linear Regression (baseline), Random Forest Regressor (100 estimators), and Gradient Boosting (100 estimators). 80/20 train-test split with random_state=42.

05

Evaluation

Model performance assessed using MAE, RMSE, and R² Score. The best-performing model (Random Forest) achieved R²=0.998 with a Mean Absolute Error of PKR 129 per trip — vs PKR 141 for Gradient Boosting and PKR 957 for Linear Regression.

06

Deployment

The trained model is embedded into this interactive web interface. User inputs are passed through the same preprocessing pipeline used during training, ensuring consistent predictions.
